# Instructions for Saving Generated Markdown Files

This file provides guidance on where to save generated Markdown (`.md`) files within the `template_knowledge/` directory. The goal is to organize knowledge effectively.

## General Guidance

When a skill generates a Markdown file, please assess its content and save it into the most appropriate subdirectory listed below. If the content spans multiple categories or if you are unsure, ask the user for clarification.

## Subfolder Mapping

*   **`career/`**: Use this folder for any content related to career planning, job applications, interview preparation, professional development, and skill tracking.
*   **`companies/`**: Store research, notes, and information about specific companies, potential employers, or industry analysis here.
*   **`daily/`**: Intended for daily logs, meeting notes, stand-up summaries, and short-term task tracking.
*   **`goals/`**: Save documents related to personal or professional goals, OKRs (Objectives and Key Results), and long-term planning.
*   **`ideas/`**: This is a catch-all for brainstorming sessions, new project ideas, feature concepts, and early-stage thoughts.
*   **`people/`**: Use for notes on individuals, networking contacts, or collaborators.
*   **`personal/`**: For general personal notes, self-reflection, learning summaries not specific to a career path, or miscellaneous information.
*   **`resumes/`**: Store different versions of resumes, cover letter templates, and any related documents.

## When in Doubt

If the category for a generated file is not immediately clear, consider saving it to `ideas/` or `personal/`, or prompt the user for their preferred location.
